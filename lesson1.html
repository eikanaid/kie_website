Build a Status Line
in Claude Code

Result — live statusline output
📁 Diana Proj/Kie · 🤖 Opus 4.7 · ⚡ normal · 🌿 main · 📊 ████░░░░░░ 40%
Key summary

A fully live statusline — all fields read from the JSON Claude Code pipes via stdin on every 5-second refresh. No env vars, no file reads, no stale data.

Key insight
💡 How it works

Claude Code pipes a live JSON object via stdin to the statusline command on every refresh. Reading from stdin means folder, model, and context are always real-time session data — no polling, no file reads.

{ "workspace": { "current_dir": "/path/to/cwd" }, "model": { "display_name": "Opus 4.7" }, "context_window": { "used_percentage": 40 } }
What it shows
📁Folder — workspace.current_dir, last 3 components, ~ for home
🤖Model — model.display_name, live — updates the moment you run /model
⚡Effort — $CLAUDE_EFFORT env var (the only field not in stdin)
🌿Git branch — git -C "$cwd", no cd, handles spaces in paths
📊Context bar — context_window.used_percentage, 10-char █░ bar
Step by step
1
Create the statusline script

Create ~/.claude/statusline.sh and paste in the script below.

#!/bin/bash # Claude Code pipes live session JSON via stdin on every refresh json=$(cat) # Parse all three fields at once parsed=$(echo "$json" | python3 -c " import json, sys, os d = json.load(sys.stdin) cwd = d.get('workspace', {}).get('current_dir', os.getcwd()) model = d.get('model', {}).get('display_name', 'unknown') pct = int(d.get('context_window', {}).get('used_percentage', 0)) print(cwd); print(model); print(pct) ") cwd=$(echo "$parsed" | sed -n '1p') model_name=$(echo "$parsed" | sed -n '2p') pct=$(echo "$parsed" | sed -n '3p') # Folder: last 3 path components, ~ for home folder=$(echo "$cwd" | awk -F'/' '{n=NF; print (n>=3 ? $(n-2)"/"$(n-1)"/"$n : $0)}' \ | sed "s|$HOME|~|") # Effort: the one field not in stdin effort=${CLAUDE_EFFORT:-?} # Git branch: -C avoids cd, handles spaces in paths branch=$(git -C "$cwd" rev-parse --abbrev-ref HEAD 2>/dev/null) [ -z "$branch" ] && branch="no-git" # Context bar: filled █ + empty ░ bar=$(python3 -c "f=round($pct/10); print('█'*f+'░'*(10-f))") echo "📁 $folder 🤖 $model_name ⚡ $effort 🌿 $branch 📊 $bar ${pct}%"

Then make it executable:

chmod +x ~/.claude/statusline.sh
2
Register it in settings.json

Open ~/.claude/settings.json and add the "statusLine" block inside the top-level object.

"statusLine": { "type": "command", "command": "bash ~/.claude/statusline.sh", "padding": 2, "refreshInterval": 5 }
3
All fields update live — nothing to maintain

Every 5 seconds Claude Code re-runs the script and pipes a fresh JSON payload. Switch models with /model, change folders, fill up the context window — the statusline reflects it all automatically within 5 seconds.

✓

You're all set

Restart Claude Code after editing settings.json — the statusline appears at the bottom of the terminal.
